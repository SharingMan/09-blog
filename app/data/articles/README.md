# 文章管理说明

每篇文章都是一个独立的 Markdown 文件，文件名格式为：`{id}.md`

## 如何添加新文章

1. 创建一个新的 Markdown 文件，例如：`6.md`
2. 文件开头必须包含前置元数据（Front Matter）：

```markdown
---
title: 文章标题
date: 2025年1月20日
readTime: 5 分钟
category: 设计
tags: 极简主义, UI设计, 用户体验
---

这里是文章内容...
```

## 元数据字段说明

- `title`：文章标题（必需）
- `date`：发布日期，格式：`2025年1月20日`（必需）
- `readTime`：阅读时长，格式：`5 分钟`（必需）
- `category`：文章分类（可选），如：`设计`、`技术`、`生活` 等
- `tags`：文章标签（可选），多个标签用英文逗号分隔，如：`极简主义, UI设计, 用户体验`

3. 在元数据下方直接写 Markdown 格式的文章内容

## 如何编辑文章

直接打开对应的 `.md` 文件进行编辑即可。

例如：
- 编辑第一篇文章：打开 `1.md`
- 编辑第二篇文章：打开 `2.md`

## 文件命名规则

- 使用数字作为文件名：`1.md`, `2.md`, `3.md` 等
- ID 必须唯一
- 建议按时间顺序递增

## Markdown 语法

支持所有标准 Markdown 语法：

- 标题：`## 二级标题`, `### 三级标题`
- 列表：`- 项目` 或 `1. 项目`
- 粗体：`**文字**`
- 斜体：`*文字*`
- 引用：`> 引用内容`
- 代码：`` `代码` `` 或 ` ```代码块``` `
- 链接：`[文字](URL)`

保存文件后，Next.js 会自动重新编译，刷新浏览器即可看到更新。
